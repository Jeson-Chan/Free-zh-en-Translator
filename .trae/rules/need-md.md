# 翻译软件开发需求说明（用于 Trae 插件开发）

请帮我开发一款桌面端翻译软件，具体要求如下：

## 1. 核心功能

- 调用 **DeepSeek API**（兼容 OpenAI 格式，推荐使用 `deepseek-reasoner` 或 `deepseek-chat` 模型）实现高质量的中英互译。
- 特别针对**学术论文与专业文献中的术语**进行精准、语境准确的翻译。

## 2. 用户界面

- 应用启动后，在屏幕右下角（或其他合适位置）生成一个**可拖拽的小型悬浮窗**。
- 悬浮窗默认显示，但支持通过点击按钮或快捷键**隐藏/显示**。
- 悬浮窗内包含：
  - 一个多行文本输入框；
  - 一个“翻译”按钮；
  - 一个区域用于显示翻译结果。
- 支持粘贴剪贴板内容（例如从 PDF 复制的英文段落）并一键翻译。

## 3. API 集成

- 使用 DeepSeek 的 v3.2 模型接口。
- 在请求的 `messages` 中明确指示模型：
  > “你是一位专业的学术翻译专家，请将以下英文（或中文）内容准确翻译为中文（或英文），保留专业术语的准确性，并保持原文格式。”
- 支持用户配置自己的 DeepSeek API Key：
  - 可通过配置文件（如 `config.json`）或首次启动时弹出设置窗口完成。

## 4. 技术栈建议

- **Python + PyQt5**（推荐，轻量且适合系统级悬浮窗）
- 确保跨平台兼容性（Windows / macOS）

## 5. 附加功能（可选）

- 自动检测输入语言（中/英）；
- 保存最近 10 条翻译历史；
- 快捷键呼出/隐藏悬浮窗（例如 `Ctrl+T`）；
- 点击翻译结果自动复制到剪贴板。

## 输出要求

请生成：
- 完整的项目目录结构；
- 核心代码模块（特别是悬浮窗 UI 实现和 DeepSeek API 调用逻辑）；
- 必要的依赖说明（如 `requirements.txt` 或 `package.json`）。
